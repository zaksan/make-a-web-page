Lesson 2:
Creating a Structured Doc with HTML

Get in line, block!

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Boxy, but Good
  1. Content. It’s the nucleus. The other layers serve to protect so no overlap or sit too close.
  2. Then padding. The padding clears an area around the content. Affected by background color of the box. Its the space inside the box that protects the content.
  3. Border goes around padding and content. Inherited from color of box.
  4. Margin clears an area around the border. It’s the space between boxes. Transparent.

This CSS is your friend: "outline: 1px solid red;"

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. Your mother... yeah, that's right. Your mother!
In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

The curious relationship between indented HTML and his neighbors

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

CSS. Like a pit of despair, but better documented.

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.
